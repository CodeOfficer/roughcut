# Keyboard Shortcuts

Comprehensive guide to keyboard navigation and shortcuts for RevealJS presentations generated by GenAI Tutorial Factory.

---

## Overview

All presentations generated by this project support RevealJS keyboard shortcuts for interactive navigation. These shortcuts work in:
- **Manual mode** (`npm run dev`) - Full keyboard control
- **Auto mode** (`npm run dev:auto`) - Shortcuts available but auto-navigation active
- **HTML presentations** - When opened directly in browser

> **Note**: During video recording, keyboard navigation is handled programmatically via Playwright.

---

## Default Keyboard Shortcuts

### Navigation

| Key | Action | Description |
|-----|--------|-------------|
| `→` `↓` `Space` `N` `L` `J` | Next slide | Advance to the next slide |
| `←` `↑` `P` `H` `K` | Previous slide | Go back to the previous slide |
| `Home` | First slide | Jump to the first slide |
| `End` | Last slide | Jump to the last slide |

**Tip**: Multiple keys are provided for the same action to accommodate different user preferences (arrow keys, vim-style, etc.).

### Feature Shortcuts

| Key | Action | Description |
|-----|--------|-------------|
| `ESC` or `O` | Toggle overview mode | View all slides in a grid |
| `F` | Toggle fullscreen | Enter/exit fullscreen mode |
| `S` | Speaker notes | Open speaker notes in new window |
| `V` or `B` or `.` | Pause (blackout) | Pause and blank the screen |
| `?` | Show help | Display keyboard shortcuts overlay |

---

## Overview Mode (`ESC` or `O`)

Press `ESC` or `O` to see all slides in a grid layout:
- Navigate with arrow keys
- Click a slide to jump to it
- Press `ESC` again to return to presentation

**Use cases**:
- Quick navigation to any slide
- Visual overview of entire presentation
- Useful during Q&A sessions

---

## Speaker Notes (`S`)

Press `S` to open speaker notes in a separate window:
- Shows current slide and next slide preview
- Displays speaker notes (if `@notes:` directive used)
- Shows presentation timer
- Syncs navigation between windows

**Setup**:
1. Add `@notes: Your notes here` to slides in markdown
2. Run presentation
3. Press `S` to open speaker view
4. Place speaker view on separate display

---

## Fullscreen Mode (`F`)

Press `F` to toggle fullscreen:
- Hides browser UI for cleaner presentation
- Works across all major browsers
- Press `F` or `ESC` to exit

## Customizing Keyboard Shortcuts

### Method 1: Via Config (Recommended for Phase 2)

Once config support is fully implemented in Phase 2, you'll be able to customize keyboard shortcuts in frontmatter:

```yaml
---
title: My Presentation
theme: black
config:
  keyboard:
    13: 'next'    # ENTER goes to next slide
    32: null      # Disable SPACE key
    84: 'toggleOverview'  # T for overview
---
```

### Method 2: Direct HTML Modification (Advanced)

For advanced users, you can modify the generated HTML to add custom keyboard bindings:

```javascript
Reveal.initialize({
  // ... other config
  keyboard: {
    84: () => startTimer(),  // T key starts timer
    82: () => resetTimer()   // R key resets timer
  }
});
```

## Common Key Codes Reference

Use these key codes when customizing keyboard bindings (Phase 2+):

| Key | Code | Key | Code | Key | Code |
|-----|------|-----|------|-----|------|
| ENTER | 13 | ESC | 27 | SPACE | 32 |
| ← | 37 | ↑ | 38 | → | 39 |
| ↓ | 40 | Home | 36 | End | 35 |
| A-Z | 65-90 | 0-9 | 48-57 | | |

**Action types**:
- **Function**: `84: () => { /* code */ }` - Execute custom function
- **String**: `13: 'next'` - Call RevealJS API method
- **null**: `32: null` - Disable key

## Accessibility

### Alternative Navigation

RevealJS provides multiple keys for the same action to improve accessibility:
- **Arrow keys**: Standard navigation
- **Vim-style (H/J/K/L)**: For users comfortable with vim
- **N/P keys**: Next/Previous alternative

### Screen Reader Compatibility

When using presentations with screen readers:
- All slides have proper semantic HTML
- Navigation is announced to screen readers
- Slide numbers and titles are accessible

## Tips & Best Practices

### During Presentations

1. **Use overview mode** (`ESC`) to quickly navigate to any slide
2. **Fullscreen mode** (`F`) for distraction-free presentations
3. **Speaker notes** (`S`) for private notes and timing
4. **Pause** (`V` or `B`) when answering questions

### For Video Recording

During automated video recording (`npm run tutorial:full`):
- Keyboard navigation is disabled
- Playwright controls all navigation programmatically
- Timing is synchronized with audio narration

### For Interactive Presentations

When creating presentations with interactive elements:
- Consider disabling certain keys to prevent accidental navigation
- Use custom keyboard bindings for interactive features (Phase 2+)
- Test navigation with actual users

## Troubleshooting

### Keyboard shortcuts not working

**Possible causes**:
1. Browser window doesn't have focus - Click on presentation first
2. Interactive element has focus - Click outside the element
3. Browser fullscreen mode interfering - Exit and re-enter fullscreen

**Solutions**:
- Click anywhere on the slide to ensure focus
- Press `ESC` to reset focus
- Reload the page if shortcuts stop responding

### Help overlay not showing

Press `?` to display the help overlay. If it doesn't appear:
1. Ensure RevealJS initialized correctly
2. Check browser console for errors
3. Try reloading the presentation
